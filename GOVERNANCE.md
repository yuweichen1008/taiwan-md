# 🏛️ Taiwan.md 社群治理

## Contributor Ladder（貢獻者階梯）

Taiwan.md 歡迎所有人參與，從閱讀到維護都是貢獻。

### 🌱 Reader（讀者）

- 閱讀、分享、在社群推薦
- 透過 Issue 回報錯誤或建議新題目
- **不需要任何技術背景**

### 🌿 Contributor（貢獻者）

- 提交 PR（文章、翻譯、修正、工具）
- 至少 1 個 PR 被 merge
- 名字會出現在 README Contributors 區
- **如何開始：** 找一個 `good-first-article` issue，寫一篇文章

### 🌳 Reviewer（審核者）

- 至少 3 個高品質 PR 被 merge
- 被邀請加入 review 團隊
- 可以 approve PR（但 merge 權仍在 Maintainer）
- 負責特定分類的內容品質

### 🏔️ Maintainer（維護者）

- 長期穩定貢獻 + 對特定領域有專業知識
- 擁有特定分類目錄的 merge 權
- 參與 Editorial Board 討論
- 每月至少 review 2 個 PR

---

## Editorial Board（編輯委員會）

處理以下事項：

- 爭議性內容的審核（政治、歷史詮釋等）
- 新分類的建立
- 編輯方針（EDITORIAL.md）的修訂
- 重大架構變更

**目前成員：** @frank890417（founder）

**申請加入：** 開一個 Issue，說明你的專長領域和貢獻意願。

## 內容品質標準

所有文章必須符合 [EDITORIAL.md](./EDITORIAL.md) 的規範：

1. **事實準確** — 所有數據必須標注來源
2. **多元觀點** — 涉及爭議議題需呈現不同立場
3. **策展式寫作** — 不是百科全書式的乾燥條列，要有觀點和脈絡
4. **參考資料** — 每篇文章末尾必須有 `## 參考資料`
5. **禁止塑膠句式** — 詳見 Issue #50
